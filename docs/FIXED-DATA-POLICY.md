# Fixed Data Policy

## 保護対象

敷地A〜E、辺長、建物中心・寸法、敷地・宅地面積、緯度・経度、園路、輪作区画、果樹・植物・施設の座標を固定データとして扱います。

## 変更ルール

1. 表示品質、素材、季節、成長年数、PLAN、REAL/PLANの変更を理由に固定データを書き換えません。
2. 固定データを変更する場合は、測量成果、設計図、現地確認などの根拠をPRへ記録します。
3. 変更前後の座標・寸法を比較し、意図した差分だけであることを確認します。
4. `data/fixed-site-data.js`の再帰的な`Object.freeze`を維持します。

## 起動時検証

起動時に敷地点数、建物寸法、敷地面積、宅地面積、必須データ、凍結状態を検証します。異常は`console.error`と画面上の警告で通知します。
